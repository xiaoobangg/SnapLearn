## 文档管理

- [feat] 语雀式树形编辑器：行内创建文件夹/文档（Enter 确认 / Escape 取消），替代 browser `prompt()`
- [feat] 右键菜单：新建文档/文件夹（同级或子级），快捷键提示
- [feat] 拖拽移动：HTML5 拖拽将文档/文件夹移到任意文件夹下，`PUT /{id}/move` 接口
- [feat] 自动保存：3 秒无操作后自动保存，工具栏显示保存状态
- [feat] md-editor-v3 替换手写正则渲染：工具栏 + 分屏预览 + 目录 + 语法高亮
- [feat] 图片上传：工具栏插图片 → 粘贴/选择 → `POST /upload-image`，存入 `uploads/images/`
- [feat] 批量导入 MD：面板头新增导入按钮 → `POST /import`
- [feat] 可见性切换：私有/已共享一键切换，立即提交，树节点显示共享标识
- [feat] 三列折叠布局：左侧文档树 + 中间编辑器 + 右侧 AI 面板，右侧可折叠
- [feat] AI 对话 @ 文档引用：输入 `@` 弹出文档列表，选中后注入文档名
- [feat] AI 对话历史管理：下拉菜单选择会话，每项带删除按钮，默认加载最新会话
- [fix] 树形结构失效：Jackson SNAKE_CASE 导致 `parent_id`/`doc_type` 前端始终 undefined → `normalizeDoc()` 转换
- [fix] 图片上传路径错误：`file.transferTo()` 相对路径 → `.toAbsolutePath()`
- [fix] AI 对话换行丢失：SSE 解析 `isFirstLine` 标志还原多行
- [fix] 侧边栏菜单溢出：main-menu 独立滚动

## 多 Agent AI

- [feat] 主 Agent + 编辑/检索子 Agent 架构：ChatClient 模式升级为 ReactAgent 模式
- [feat] 主 Agent (DocumentMasterAgent)：意图路由 → `callEditAgent` / `callChatAgent`
- [feat] 编辑子 Agent：ChatClient + 写工具集 (createDocument, createFolder, updateDocument, updateDocumentByName, appendDocument, batchCreateStructure, getDocument, listDocuments)
- [feat] 检索子 Agent：ChatClient + 读工具集 (searchDocuments, getDocument, listDocuments)
- [feat] 批量创建：`batchCreateStructure` 一次性创建多文件夹+文档，AI 自行分类，无需逐项确认
- [feat] 按名称修改：`updateDocumentByName` 直接传文档名，内部自动查 ID
- [feat] 文件夹创建：`createFolder` 工具，docType 正确设为 folder
- [feat] PostgresSaver 会话记忆：主 Agent checkpoint 持久化，跨轮对话连贯
- [feat] 子 Agent 无状态：ChatClient 临时创建，userId 通过 AgentContext.setUserId() ThreadLocal 传递
- [feat] 博客 AI ChatClient：纯内存 ChatMemory + MessageChatMemoryAdvisor，区别于管理端 JDBC 持久化
- [fix] 子 Agent userId 丢失：`AgentContext.setUserId()` 同步设置 ThreadLocal
- [fix] Tool 导入：`org.springframework.ai.tool.annotation.Tool` / `ToolParam`

## 一句话总结

**文档管理**：语雀式树形编辑器，支持行内创建、拖拽移动、自动保存、Markdown 实时预览、图片上传、可见性切换。右侧 AI 面板集成多 Agent（主 Agent 路由 → 编辑子 Agent / 检索子 Agent），支持 @ 文档引用和批量创建。

**博客系统**：公开免登录访问，三列布局（文档树 + 文章列表 + AI 对话），支持评论、MdPreview 渲染、向量检索 RAG，未登录只显示博客入口。

**多 Agent AI**：主 Agent（ReactAgent + PostgresSaver 记忆）+ 两个子 Agent（ChatClient 无状态），编辑子 Agent 具备 8 个工具（含批量创建、按名修改），检索子 Agent 具备 3 个只读工具。

**基础设施**：3 个数据库迁移（用户统一、博客评论、文档树）、向量元数据增强、公共 SSE 流式接口、侧边栏登录态感知。
